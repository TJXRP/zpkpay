SDK Reference

The ZPK API is a REST interface to the AIFS ShatterEngine. Your application calls these endpoints to shatter payloads, settle anchors on Xahau, and reassemble data for authorized parties.

Engine boundary. The ShatterEngine is proprietary and never exposed. All endpoints return results only -- no engine internals are observable from the API layer.

Base URL

https://api.aifs.dev/v1

All requests must use HTTPS. No authentication is required for the public demo API. Production pilots use an issued API key passed as X-API-Key header.

POST /liquify

POSThttps://api.aifs.dev/v1/liquify

Shatters a payload into 1,024 entropic particles and returns a DNA anchor. The anchor is the only artifact your application needs to proceed to settlement.

Request body

ParameterTypeRequiredDescription
originAddressstringrequiredSender's Xahau/XRPL wallet address. Used as the XOR identity salt.
receiverAddressstringrequiredRecipient's wallet address. Stored as a SHA-256 hash -- never as a raw address.
metadatastringrequiredThe payload to shatter. Plain text, JSON string, or ISO 20022 XML string.
allowSenderReassemblybooleanoptionalIf true, the sender may also reassemble the payload. Default: false.

Response

FieldTypeDescription
dnastring24-character uppercase hex DNA anchor. Pass this to Xahau as the HookParameterValue.
statusstring"shattered" on success.
// Request fetch('https://api.aifs.dev/v1/liquify', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ originAddress: 'rSenderWalletAddress...', receiverAddress: 'rReceiverWalletAddress...', metadata: '...', allowSenderReassembly: true }) }); // Response { "dna": "A3F8B2C91D4E7F60A1B2C3D4", "status": "shattered" }

POST /materialize

POSThttps://api.aifs.dev/v1/materialize

Reassembles a shattered payload for an authorized party. The engine verifies the wallet address against the stored identity salt and returns the original payload only if authorization passes.

Request body

ParameterTypeRequiredDescription
dnastringrequiredThe 24-character DNA anchor returned by /liquify.
walletAddressstringrequiredThe requesting party's wallet address. Must match sender or receiver.
rolestringoptional"sender" or "receiver". Defaults to "receiver".

Response

FieldTypeDescription
payloadstringThe original payload as submitted to /liquify.
statusstring"reassembled" on success.
fetch('https://api.aifs.dev/v1/materialize', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ dna: 'A3F8B2C91D4E7F60A1B2C3D4', walletAddress: 'rReceiverWalletAddress...', role: 'receiver' }) });

POST /anchor-status

POSThttps://api.aifs.dev/v1/anchor-status

Checks whether a DNA anchor has been settled on Xahau and confirmed by the Hook. Use this to verify settlement before allowing reassembly in gated flows.

Request body

ParameterTypeRequiredDescription
dnastringrequiredThe DNA anchor to check.
// Response (settled) { "settled": true, "txid": "AB12CD34...", "ledger": 1234567 } // Response (pending) { "settled": false, "txid": null }

ZPK-Enforcer Hook

The ZPK-Enforcer Hook is a Xahau Hook deployed on Mainnet. Install it on any Xahau receiving address to enforce that only payments carrying a valid ZPK DNA anchor are accepted.

Hook live on Xahau Mainnet. SetHook transaction: 39249C41. The Hook hash is publicly installable -- any Xahau address can use it.

Hook installation

Install the Hook on your receiving address using the Xahau Hooks Builder or any Xahau-compatible SDK.

Hook hash

9D35CFCE2ACF4C59E15063AD09119AC4269EA1E11766F6A412D2B1443BDBBF6A

SetHook transaction (xrpl.js)

{ TransactionType: 'SetHook', Account: 'rYourReceivingAddress...', Hooks: [{ Hook: { HookHash: '9D35CFCE2ACF4C59E15063AD09119AC4269EA1E11766F6A412D2B1443BDBBF6A', HookOn: '0000000000000000', HookApiVersion: 0, CreateCode: '', Flags: 1 } }] }

HookParameter format

Every payment to a ZPK-enforced address must include the DNA anchor as a HookParameter.

FieldValue
HookParameterName444E41 (hex encoding of "DNA")
HookParameterValueDNA anchor hex-encoded to ASCII hex. Example: anchor A3F8... becomes 413346382E2E2E
// Build the HookParameter from a DNA anchor const dna = 'A3F8B2C91D4E7F60A1B2C3D4'; const hookParamValue = Buffer.from(dna, 'utf8').toString('hex').toUpperCase(); const hookParameters = [{ HookParameter: { HookParameterName: '444E41', HookParameterValue: hookParamValue } }];

Error codes

CodeHTTPDescription
ZPK_ANCHOR_INVALID400The DNA anchor failed engine signature verification. Rejected before Hook submission.
ZPK_NOT_AUTHORIZED403The wallet address does not match the sender or receiver stored for this anchor.
ZPK_NOT_FOUND404No particle set found for the provided DNA anchor.
ZPK_ALREADY_SETTLED409This anchor has already been settled on Xahau. Single-use replay protection enforced.
ZPK_ENGINE_ERROR500Internal engine error. Retry with exponential backoff.

DNA anchor format

Every DNA anchor is exactly 24 uppercase hex characters (96 bits).

A 3 F 8 B 2 C 9 1 D 4 E 7 F 6 0 A 1 B 2 | C 3 D 4 <─────────── 20 chars ───────────────────> <─ 4 ─> SHA-256 particle hash engine (truncated) signature (HMAC-SHA256)

The last 4 characters are an HMAC-SHA256 engine signature. Any anchor submitted to the API that does not carry a valid engine signature is rejected with ZPK_ANCHOR_INVALID before it reaches the Hook.
